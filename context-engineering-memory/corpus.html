The corpus

Sixteen sources on context engineering and agent memory: twelve arXiv papers, three industry engineering writeups, and one ICML paper. Every citation chip on this site resolves to an entry here, and each entry links onward to the original on arXiv or the publishing lab. That chain — claim → chip → bibliography → source — is the whole point.

◇ How to read this page

Entries are grouped by the role each plays in the thread. The cite label in each entry is the exact chip used across the site. Dates and venues are taken from the sources themselves; where a source did not state a venue or date, this is noted. The 2026 arXiv papers are preprints under review, not peer-reviewed; the three lab writeups are industry engineering guidance, not controlled studies. Treat both accordingly.

Anchor — the field-defining surveys

AnchorA Survey of Context Engineering for Large Language Models
arXiv:2507.13334 · Mei, Yao, Ge, Wang et al. (Inst. of Computing Technology, CAS; UC Merced; Queensland; Peking; Tsinghua), Jul 2025 · survey · cite: Mei+ '25
Names and formalizes "context engineering" as a discipline that "transcends simple prompt design," reconceiving context as a dynamically assembled set of components C = A(c₁…cₙ) optimized under |C| ≤ L_max. Synthesizes 1400+ papers into Foundational Components (retrieval, processing, management) and System Implementations (RAG, memory, tools, multi-agent), and names the field's defining gap: a comprehension-vs-generation asymmetry.

AnchorMemory in the Age of AI Agents
arXiv:2512.13564 · Hu, Liu, Yue, Zhang et al. (NUS, Renmin, Fudan, Peking, NTU, Oxford, OPPO et al.), Dec 2025 · survey · cite: Hu+ '25
Argues the long/short-term dichotomy is insufficient and proposes a unified "Forms–Functions–Dynamics" taxonomy of agent memory. Explicitly separates context engineering ("resource allocation and interface correctness") from agent memory ("a persistent, self-evolving cognitive state"), locating their overlap at working memory — the conceptual hinge for this whole review.

Core — methods, systems, surveys & benchmarks

CoreMemory for Autonomous LLM Agents: Mechanisms, Evaluation, and Emerging Frontiers
arXiv:2603.07670 · Pengfei Du (Hong Kong Research Inst. of Technology), Mar 2026 · survey · cite: Du '26
Formalizes agent memory as a write–manage–read loop and a POMDP belief state, organized along temporal scope × representational substrate × control policy. Coins useful failure vocabulary (summarization drift, attentional dilution, self-reinforcing error, schema drift) and crystallizes the cross-benchmark lesson that "long context is not memory."
arXiv abstract ↗
CoreAgentic Context Engineering: Evolving Contexts for Self-Improving Language Models (ACE)
arXiv:2510.04618 · Q. Zhang, Hu, Upasani et al. (Stanford, SambaNova, UC Berkeley), ICLR 2026 · framework + empirical · cite: Zhang+ '25 (ACE)
Treats context as an evolving "playbook" of itemized strategies, updated by a Generator/Reflector/Curator loop with incremental delta edits. Diagnoses two failure modes of iterative context curation — brevity bias and context collapse — and reports +10.6% (agents) / +8.6% (domain) with large adaptation-cost reductions.
arXiv abstract ↗
CoreMem0: Building Production-Ready AI Agents with Scalable Long-Term Memory
arXiv:2504.19413 · Chhikara, Khant, Aryan, Singh, Yadav (mem0.ai), Apr 2025 (ECAI 2025) · system + empirical · cite: Chhikara+ '25 (Mem0)
A production memory architecture that extracts, consolidates (ADD/UPDATE/DELETE/NOOP) and retrieves salient facts, with a graph variant Mem0ᵍ. On LOCOMO it reaches near-full-context accuracy at ~91% lower p95 latency and >90% lower token cost — the canonical "memory beats brute-force context on cost" datapoint.
arXiv abstract ↗
CoreAgentic Memory: Learning Unified Long-Term and Short-Term Memory Management for LLM Agents (AgeMem)
arXiv:2601.01885 · Yu, Yao, Xie, Tan, Feng, Li, Wu (Wuhan University, Alibaba Group), Jan 2026 · framework + empirical · cite: Yu+ '26 (AgeMem)
Exposes six memory operations (LTM: add/update/delete; STM: retrieve/summarize/filter) as tool actions in the agent's own policy, trained end-to-end with a three-stage curriculum and step-wise GRPO. Turns context curation from a heuristic schedule into a learned control problem with delayed reward.
arXiv abstract ↗
CoreMemSkill: Learning and Evolving Memory Skills for Self-Evolving Agents
arXiv:2602.02474 · H. Zhang, Long, Bao, Wang, Feng, W. Zhang, Yue (NTU, UIUC, UIC, Tsinghua), Feb 2026 · framework + empirical · cite: Zhang+ '26 (MemSkill)
Elevates memory operations into a learnable, evolvable "skill bank": an RL-trained controller selects skills, an executor applies them span-by-span, and a designer evolves the skill set from hard cases. Span-level extraction reaches the best quality–cost frontier (215 vs 1288 LLM calls for a strong baseline).
arXiv abstract ↗
CoreLearning Query-Aware Budget-Tier Routing for Runtime Agent Memory (BudgetMem)
arXiv:2602.06025 · H. Zhang, Yue, Feng, Long, Bao, Jin et al. (NTU, UIUC, Tsinghua, SYSU, HKUST-GZ), Feb 2026 · system + empirical · cite: Zhang+ '26 (BudgetMem)
Defers memory computation until a query arrives, exposing Low/Mid/High "budget tiers" on each pipeline module and training a lightweight RL router to pick tiers query-aware. Traces a controllable accuracy–cost Pareto frontier that envelops fixed baselines — memory extraction as a tunable inference-time knob.
arXiv abstract ↗
CoreA Human-Inspired Reading Agent with Gist Memory of Very Long Contexts (ReadAgent)
arXiv:2402.09727 · Lee, Chen, Furuta, Canny, Fischer (Google DeepMind), ICML 2024 · system + empirical · cite: Lee+ '24 (ReadAgent)
Paginates long text into episodes, compresses each into a fuzzy "gist," and interactively looks up verbatim pages on demand — extending effective context 3.5–20× while beating retrieval and full-context baselines. A canonical hierarchical-memory / compaction architecture for long-context reading.
arXiv abstract ↗ DeepMind page ↗
CoreThe Missing Memory Hierarchy: Demand Paging for LLM Context Windows
arXiv:2603.09023 · Tony Mason (UBC; Georgia Tech), Mar 2026 · system + empirical · cite: Mason '26
Argues the context window "is not memory — it is L1 cache," and applies 1960s OS abstractions (demand paging, working sets, eviction) directly. A 857-session production corpus shows 21.8% structural token waste and 84.4× median reprocessing of stale results, and a deployed proxy (Pichay) recovers context. Introduces the "inverted cost model."
arXiv abstract ↗
CoreContextual Memory Virtualisation: DAG-Based State Management and Structurally Lossless Trimming for LLM Agents
arXiv:2602.22402 · Cosmo Santoni (Imperial College London), Feb 2026 · system + empirical · cite: Santoni '26 (CMV)
Treats the conversation log as version-controlled state: a DAG of snapshots/branches ("Git for context") plus a three-pass "structurally lossless" trimmer that strips mechanical bloat while preserving every user/assistant message verbatim. Trimming cuts tokens by a mean 20% (max 86%) across 76 sessions; the dominant value is avoiding context rebuilding.
arXiv abstract ↗
CoreSideQuest: Model-Driven KV Cache Management for Long-Horizon Agentic Reasoning
arXiv:2602.22603 · Kariyappa, Suh (NVIDIA), Feb 2026 · system + empirical · cite: Kariyappa & Suh '26
Lets the model do its own KV-cache "garbage collection": an auxiliary thread on the shared context reasons about which tool-output tokens are stale and evicts them, with zero permanent token overhead to the main window. Cuts peak token utilization 56–65% and lifts serving throughput 83.9% for ≤5% accuracy loss.
arXiv abstract ↗
CoreMemoryArena: Benchmarking Agent Memory in Interdependent Multi-Session Agentic Tasks
arXiv:2602.16313 · He, Wang, Zhi, Hu et al. (Stanford, UCSD, UIUC, Princeton, Pitt), ICML 2026 · benchmark · cite: He+ '26 (MemoryArena)
Couples memorization and action in multi-session tasks whose later subtasks are underspecified unless the agent distills and reuses earlier experience. Systems near-saturated on LoCoMo fail here (near-zero success on two of four environments); external memory and RAG give no consistent gain over raw long context. The sobering reality check.

Support — industry practitioner guidance

SupportEffective Context Engineering for AI Agents
anthropic.com/engineering · Anthropic Applied AI (Rajasekaran, Dixon, Ryan, Hadfield et al.), Sep 29 2025 · engineering blog / position · cite: Anthropic '25
Anthropic's field-defining practitioner piece: context is a finite resource subject to "context rot" and a finite "attention budget," and the discipline is finding the smallest set of high-signal tokens. Popularizes just-in-time retrieval, compaction, structured note-taking, and sub-agent context isolation. Industry guidance, not a controlled study.

SupportContext Engineering: Memory, Compaction, and Tool Clearing
platform.claude.com/cookbook · Isabella He, Anthropic (Claude Cookbook), Mar 20 2026 · technical how-to · cite: Anthropic '26 (Cookbook)
Operationalizes three composable, first-party primitives — compaction (whole-window summary), tool-result clearing (drop re-fetchable payloads), and the file-backed memory tool (cross-session) — with a workload-to-primitive decision framework and illustrative token measurements from a research-agent demo. The sharpest articulation of "which primitive for which problem."
Read at platform.claude.com ↗
SupportContext Engineering for Personalization: State Management with Long-Term Memory Notes
developers.openai.com/cookbook · Emre Okcular, OpenAI (Cookbook / Agents SDK), Jan 2026 · technical how-to · cite: OpenAI '26
A reference implementation of long-term memory as structured "notes" over a local-first state object: distill candidate notes during a run, consolidate (dedup + conflict-resolve + forget) asynchronously at session end, and inject curated slices under explicit precedence rules. Argues "forgetting is not a bug — it is essential" and frames injected memory as an attack surface.
