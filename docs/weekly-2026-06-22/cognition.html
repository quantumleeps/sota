01 · Cognition

What the agent knows, remembers, and retrieves

Four papers in this sample sit at the agent's "inner loop" — how it stores experience, how it represents its own uncertainty, whether it builds a model of an environment through interaction, and whether the retrieval machinery it runs is worth its cost. They were written independently, but they rhyme: in each, a deliberately simpler, controlled design matches or beats the more elaborate one, and the residual difficulty is the long, multi-step trajectory.

Memory as write-control, not accumulation

The default design for long-term agent memory is to "remember everything" and retrieve later. Chen+ '26 argues that under real inference-cost and context budgets this produces memory bloat — irrelevant trivia crowding out what matters at retrieval time — and that the agent should instead exert write control, learning a role-specific Memory Policy that keeps only what each user cares about.

▦ Less memory, more accuracy Empirical

AdaMem improves QA accuracy by up to +9.0% over a uniform-extraction baseline (85.2% vs 76.2%) while shrinking memory volume 9.1% (371 vs 430 stored memories) Chen+ '26. Gains concentrate in "soft preference" categories — Emotion/Attitude +15.9, Schedule/Time +9.9 — and are near-zero for hard facts (Fact/Number +1.1), exactly where a policy of selective retention should help. Empirical

The paper's most useful diagnostic separates two abilities an agent needs: Definition policy adherence (following a given memory policy) versus policy inference (working out the right one from feedback). Adherence is already high (Policy-Following Rate 80–86%); the bottleneck is inference under weak signal — under explicit feedback the policy converges to ~88% ground-truth alignment, but under implicit feedback learning stalls at 53–63% Chen+ '26.

an agent should not remember everything, but instead learn what to remember, because what counts as important is subjective and role-dependent — AdaMem Chen+ '26, §Introduction

⟡ My inference — not from the papers

AdaMem's adherence-vs-inference split is, to me, the more transferable contribution than the headline accuracy number (which rests on a synthetic 5-story / 10-week benchmark with an LLM judge). "The model follows policies fine; it can't reliably infer them from weak feedback" is a clean way to state where a lot of personalization work actually fails, and it rhymes with the clarification result below — both are about an agent's difficulty inferring an unstated goal from indirect signal. I read that as one problem wearing two hats, not two findings.

Splitting confidence so the agent can ask

A single confidence scalar can't tell "I'm unsure my action is right" apart from "I'm unsure what you even want." Matsnev '26 decomposes it into action confidence and request uncertainty, giving the agent a dedicated channel for goal ambiguity that triggers a clarifying question — all prompt-only, so it runs in one forward pass on a black-box API with no logprobs or labelled trajectories.

▦ A clarification channel works — but isn't free Empirical

The decomposition improves clarification F1 by 73% over ReAct+UE and 36% over UAM, averaged across five 2026 backbones Matsnev '26. But it comes bundled with a capability dilution: mean task success falls monotonically 28.6% → 27.8% → 27.0% as each extra uncertainty objective is added to the prompt, concentrated on the hardest benchmarks (REAL 32.1% → 19.2%). The author attributes this to a fixed shared reasoning budget, not phrasing. Theory

The sharpest methodological warning is about how trajectory confidence is aggregated. Product-aggregation looks like a strong fault detector (ROC-AUC up to 0.99) — but confidence-free surrogates (uniform random, or simply 1/length) hit 0.92–0.99 too, so the signal is largely a trajectory-length proxy, not a measure of confidence Matsnev '26.

high product-aggregation ROC-AUC … can simply be evidence that the agent took more steps to fail — Uncertainty Decomposition Matsnev '26, §7.4

Does the agent build a world model — or pattern-match?

Menaged+ '26 turns a classic theory problem into an agent benchmark: a tool-calling agent must uncover a hidden deterministic finite automaton through membership and equivalence queries, with the exact algorithms L* and TTT as 100%-solving baselines and passive learners used to diagnose why a run failed.

~20%
Non-informative queries even the most consistent model issues after ~60 steps; classic learners: 0% by construction. Menaged+ '26

Two things make this more than a leaderboard. First, the paper rules out the obvious deflationary explanation — models are not imitating the classic algorithms: no run reproduces L*/TTT's query sequence, and the best model exceeds the equivalence-query bound in 92.5% of instances and produces non-monotonic hypothesis sequences throughout Menaged+ '26. Second, its failure taxonomy is mechanistic: Definition a planning failure is one where no passive learner could recover the DFA from what the agent gathered (it didn't collect enough), while a reasoning failure is one where a passive learner could (the information was there, unused). Weaker models are dominated by planning failures (Llama-3.3-70B: 78.75% of instances).

Dissecting agentic RAG: which complexity earns its keep?

Agentic RAG adds query decomposition, adaptive retrieval routing, and deep iterative loops on the assumption the complexity helps. Shaikh '26 runs a controlled, significance-tested ablation on a local Qwen2.5-7B over HotpotQA to test that assumption — and most of it doesn't survive.

The mechanism: running a short retrieval loop at all is the load-bearing move (iterative evidence accumulation), while adaptive routing actively hurts — the rule-based router fires on named entities/dates/numbers, which appear in nearly every multi-hop sub-question, sending 72% of all retrieval calls to BM25 Shaikh '26. The per-strategy scores that appear to favour BM25 are flagged by the author as a selection-bias artifact: the heuristic routes the structurally simpler, entity-anchored queries there.

most of the gain comes from running a short retrieval loop, not from adaptive routing or from many iterations — Dissecting Agentic RAG Shaikh '26, Abstract

◇ What these four share — and what limits each

Each result is a single setting: AdaMem on a synthetic 5-story benchmark with an LLM judge; the clarification study at 100 tasks per cell, single-run, on synthetic underspecification; the automata work on binary-alphabet DFAs; the RAG ablation on one model family and one benchmark with its specific two-hop structure. None licenses a field-level law. What they jointly suggest — and only suggest — is that in this corner of the week, the marginal complexity layered onto agents (uniform memory, scalar confidence, deep adaptive retrieval) was easier to add than to justify.
