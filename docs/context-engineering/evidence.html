Does codified context actually work?

The corpus reports real, mostly positive effects — but they vary enormously in rigour, from one tightly controlled isolation study to several single-repo, small-N case studies. This page lays out the measured results and then reads them honestly: the direction of the evidence is consistent and encouraging; the strength is uneven.

The strongest single result, in context

The AGENTS.md isolation study deserves its own callout because it is the only result in the corpus that cleanly isolates the causal effect of one context artifact: same task, same repo snapshot, agent run with the file present vs removed, in fresh containers.

The authors are careful about what it does not show. The token benefit concentrates in output tokens and a few very expensive runs — input/cached/total token medians barely moved (some slightly up). There is no correctness evaluation, only a 50-task manual sanity check that outputs were non-trivial; only one model family was tested; and the mechanism (less exploratory navigation) is "speculated," not yet traced Lulla+ '26. So: faster and cheaper, on small tasks, for one agent — convincingly; "better code" — not measured here.

Reading the ledger honestly

⟡ My appraisal of the evidence base

Direction is consistent; rigour is not. Across very different methods — a controlled PR study, benchmark ablations, an industrial case, multi-agent interventions — every result points the same way: shaping context/scaffolding helps, often substantially, and frequently more than swapping the model. That convergence across methods is genuinely reassuring.

But notice what is missing. The headline artifact of this thread — the human-authored CLAUDE.md/AGENTS.md content studied on the previous page — has exactly one controlled efficiency study behind it and zero controlled correctness studies. The biggest correctness gains in the ledger come from scaffolding and structure (ACI, Confucius, SOPs, roles), not from prose context files. The anchor's compelling case studies are observational. The largest percentage (+206%) is a single industrial task-type with 5 scenarios. And the context-engineering survey's eye-catching figures (an "18-fold" navigation gain, "+175.96% exact match" on bug-fixing) are secondary citations from other works, not its own measurements Mei+ '25 — they should not be read as primary evidence.

My summary: that codified context helps is well-supported in direction; how much, for which tasks, and whether it improves correctness rather than just speed/cost are largely open.

◇ What would move the needle (drawn from the papers' own future-work)

  • Controlled, per-tier benchmarking of codified-context infrastructure — the anchor names this as its top-priority future work Vasilopoulos '26.
  • AGENTS.md effect across multiple agents/models and with a real correctness/quality measure, plus tracing the navigation mechanism Lulla+ '26.
  • Studies of which file properties (specificity, structure, style) drive the effect — every file study can describe content but none yet links content to agent outcomes Chatlatanagulchai+ '25b Jiang & Nam '26.
