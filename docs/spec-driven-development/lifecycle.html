02 · The Agentic SDLC

Re-drawing the lifecycle around agents

If specs are the new primary artifact, the lifecycle that consumes them has to change shape. Three papers in this corpus re-draw the SDLC explicitly: Bhati maps every classical phase to an agentic equivalent, Mandl frames a staged transformation organised around a "harness," and Li pushes spec-driven automation upstream from requirements into architecture. The common thread is that the human moves to intent, review, and approval gates while agents do the execution between them.

From "code generation" to "delegated execution under human supervision"

Bhati's survey reframes the object of study Theory Definition:

the central object of inquiry has shifted from code generation to delegated execution under human supervision — Agentic AI in the Software Development Lifecycle Bhati '26, §Abstract

The paper proposes a six-layer reference architecture — L0 Foundation Model, L1 Reasoning/Memory/Self-Reflection, L2 Agent–Computer Interface, L3 Tools & Environment, L4 Orchestration, L5 Governance & Safety — and an Agentic SDLC (A-SDLC) that maps each classical phase to an agent equivalent, where (for example) the Requirements phase becomes "intent specification; agent drafts spec" with the human supervising Proposal Bhati '26. KPIs shift accordingly — from cycle time and defect rate toward acceptance rate and escalation quality.

▦ Scaffolding, not raw model scale, drove the capability jump Empirical Theory (cited)

Bhati compiles a SWE-bench Verified trajectory rising from 1.96% (Oct 2023) to 78.4% (Apr 2026), and argues the agent–computer interface — not just bigger models — is responsible: SWE-agent's structured interface lifted resolution to 12.5% "even with the same underlying model," so "interface design for the agent matters as much as model capability" Bhati '26. These figures are compiled from other reports, not Bhati's own measurement — treat them as a cited trend, not this paper's evidence.

Bhati also raises the corpus's recurring downstream worry — that high agent throughput makes human review the binding constraint:

If an agent can produce ten plausible patches per hour, the rate-limiting resource is human review. — Bhati '26, §The Economics of Attention

The "harness": embedding, not just a model

Mandl's contribution is a pragmatic staging of adoption — AI-assisted → AI-integrated → AI-driven (agentic) — presented "not as a rigid maturity model" but as an organising logic, centred on the idea of a harness Definition Proposal:

A harness translates model capabilities into usable development workflows. — AI-driven Software Development Mandl+ '26, §Harness

The harness is the technical/organisational embedding — context provisioning, tool access, verification, permissions, logging, human approval — spanning four cross-cutting functions: AI Context, AI Generation, AI Verification, and Governance & Control. Concretely, Mandl's illustrative repository harness uses a CLAUDE.md primary instruction file, CI/CD gates, ADRs, a domain glossary, and explicit AI guardrails (the repo being "only one part" of a full harness) Mandl+ '26.

▦ AI redistributes effort; it does not uniformly reduce it Theory

Mandl argues routine creation accelerates while QA, verification, context provisioning and governance gain weight — "the amount of generated code is not a meaningful success measure" — and that gains "arise primarily where AI is embedded in stable, well-organized development processes." Where foundations are missing, "AI can amplify existing weaknesses rather than compensate for them" Mandl+ '26. The recommended measurement pairs DORA delivery metrics with AI-specific ones (test coverage, security findings, human-correction frequency, traceability, cost per agentic task).

Mandl is explicit that AI-generated requirements are a draft, not a deliverable spec: "AI-generated requirements should therefore not be treated as finished specifications, but as drafts that require systematic review and domain-specific approval" Mandl+ '26 — and treats "Zero-Touch Engineering" as a boundary case that shifts rather than removes human work.

Pushing spec-driven automation upstream: requirements → architecture

Most agentic SDLC tooling targets code; Li's MAAD targets the phase before it, turning a Software Requirements Specification into multi-view "4+1" architecture blueprints via four role-specialised agents (Analyst, Modeler, Designer, Evaluator) with RAG over architecture standards and a three-layer memory Proposal Empirical Li+ '26. Against MetaGPT on 10 SRS cases (same base LLM), MAAD produced the full requirements→model→documentation→evaluation spectrum where the baseline produced only partial artifacts — and crucially, MetaGPT scored interface complexity 0.00 across all 10 projects (no explicit interface operations), while inventing requirements not grounded in the SRS.

▦ Evaluation belongs inside generation, and the base model sets a ceiling Theory

Two of Li's framing claims travel beyond MAAD. First, "evaluation is not a post-hoc activity but an integral part of architecture generation." Second, on why orchestration is not a cure-all: "multi-agent orchestration acts as a capability amplifier… while exposing the deficiencies of weaker LLMs more apparently," so the base LLM's competence "establish[es] a performance ceiling for the entire pipeline, rather than a mere bottleneck" Li+ '26. Practitioners (n=6) judged MAAD an assistant requiring a human in the loop, not an autonomous replacement.

⟡ My inference — the corpus agrees on the skeleton, not the flesh

Bhati, Mandl, and Li describe recognisably the same skeleton — intent/spec at the top, agentic execution in the middle, human gates at the boundaries, governance as a cross-cutting layer — but none validates a full end-to-end A-SDLC. Bhati's lifecycle map is a proposal with cited (not measured) capability data; Mandl's staging rests on one exploratory case; Li's empirical result is one phase (architecture) on 10 cases. The lifecycle re-drawing is, in this corpus, a well-argued blueprint with point evidence at individual phases — not a measured pipeline. Chips mark what I reason from.
