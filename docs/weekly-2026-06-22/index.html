SotA Weekly · the agent-research frontier · 15–22 June 2026

The week the second-order questions arrived

Ten papers, one per agent subfield, all posted to arXiv in a single week. What recurs across this cross-section is not a new capability but a change of register: a striking number of these papers stop asking "can an agent do X?" and start asking "does the machinery we added to make it do X actually earn its keep — and where exactly does it break?" Fixed beats adaptive; subtask success doesn't survive the full workflow; a leaderboard rank doesn't transfer; the unaudited part of the agent is its own runtime. Read together — and only together — they read like a field auditing its own defaults.

⟡ Scope — read this first

This is a review of 10 sources, selected on 22 June 2026 from a 319-paper automated search (arXiv + OpenAlex + Hugging Face) across ~14 agent subfields, filtered to the 240 papers genuinely posted 15–22 June 2026, then curated to one representative per subfield for breadth. It is a reading of this ten-paper sample — a deliberately broad but tiny, search-shaped slice of one week of mostly un-peer-reviewed preprints — not a survey of the week and not a claim about where the field is going. The "second-order" framing above is the pattern I see across these ten; the papers did not coordinate. Where this site says "across these papers," it means exactly these ten. What recurs across these 10 papers

Threads that show up in more than one of these ten, independently — a shared register in this sample, not a field-level verdict. Each links to the page that develops it.

1 · The simpler, controlled baseline keeps matching or beating the added machinery Empirical

The loudest throughline. A controlled ablation finds fixed hybrid retrieval beats rule-based adaptive routing and that two loop iterations ≈ five Shaikh '26; learning what to remember (write-control) beats remembering everything, with more accuracy on less memory Chen+ '26; and on a hard new benchmark "simply adding an agent loop is not enough" — a wrapper can score worst of all Zou+ '26. → Cognition

2 · Aggregate scores hide what deployment exposes Empirical Proposal

Several papers drive a wedge between the headline number and the deployed behaviour. A leaderboard rank that fails to transfer out-of-sample (ρ=−0.13) is not predictive, so rank by predictive validity instead Patel+ '26; strong subtask scores collapse to 26.3% end-to-end once errors accumulate over a workflow Zou+ '26; and a product-aggregated confidence score that looks discriminative is largely a trajectory-length proxy Matsnev '26. → Evaluation

3 · The agent's real safety boundary is its runtime and its tool choices — not the model alone Definition Empirical

Three papers relocate "agent safety" away from prompt/jailbreak defence. The privileged runtime — parser, dispatcher, skill loader, permission gate — is an unaudited boundary: "even an agent that correctly refuses every dangerous prompt can still be exploited through its runtime" Zhang+ '26; agents over-select higher-privilege tools, and general safety alignment does not transfer to least-privilege choice Yang+ '26; and robustness has to be a design objective, since "workflows cannot be reliably protected by patch-style defenses alone" Yue+ '26. → Safety

4 · Whatever robustness or control the agent has must be built in, not patched on Proposal Empirical

A constructive counterpart to thread 3. The strongest-under-attack system bakes safety primitives into the design search rather than bolting on defences Yue+ '26; privilege hygiene comes from privilege-aware post-training, not prompt instructions that crumble under transient failure Yang+ '26; and memory quality comes from write-side control at extraction time, not post-hoc compression Chen+ '26. → Safety

5 · The shared ceiling is the long-horizon closed loop — reasoning models extend it, none close it Empirical Theory

Where these papers measure long trajectories, they find the same wall. Interactive automata discovery degrades to ≤25% at 8–9 states and agents emit rising fractions of non-informative queries as interactions lengthen Menaged+ '26; workflow success accumulates error to 26.3% Zou+ '26; transient tool failures erode an agent's confidence in low-privilege tools over a multi-turn episode Yang+ '26; and personalized memory retrieval decays over a 10-week horizon without write-control Chen+ '26. Reasoning models clearly help Menaged+ '26 but pay a "capability dilution" tax when loaded with extra objectives Matsnev '26. → Cognition

Map of the territory

Cognition

What agents know, remember, and retrieve: write-controlled memory, a decomposed uncertainty channel for clarification, interactive world-model inference, and a controlled dissection of agentic RAG. The cluster where "simpler, controlled" repeatedly wins.

Safety & Robustness

The agent as an attack surface: the privileged runtime layer as an unaudited boundary, over-privileged tool selection that safety-alignment doesn't fix, and automated design that makes robustness a first-class objective. Defence moves from the prompt to the substrate.

Evaluation & Serving

Measuring and running agents honestly: predictive validity over aggregate leaderboards, a feedback-driven computer-use benchmark where end-to-end success craters, and a serving scheduler that treats the agentic workload's own variability as the problem.
