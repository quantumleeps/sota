04 · Architecture

Multi-agent & the harness

Codified context does not float free; it lives inside an architecture. The corpus traces a clear arc — from role-played multi-agent frameworks (2023), through the discovery that the agent's interface and scaffold drive performance (2024), to the present view that the agent system is mostly operational harness around a thin reasoning loop (2026).

The architectural axes

The survey supplies the vocabulary: single-agent vs multi-agent, and within multi-agent, vertical (a lead agent with subordinates) vs horizontal (equals in a shared thread) Masterman+ '24 Definition. Its persona definition is the bridge to codified context: a persona is the role plus "descriptions of any tools the agent has access to" — i.e. the system prompt is codified role-and-tool context. The survey's design levers (clear leadership, role definition, message filtering, feedback phases) all "yield improved agent performance," but it cautions against assuming more agents help:

multi-agent discussion does not necessarily enhance reasoning when the prompt provided to an agent is sufficiently robust.— Landscape of AI Agent Architectures Masterman+ '24, Key Findings

It also reports the concrete value of structure: a designated leader makes teams ≈10% faster and shifts communication from agents ordering each other around to the leader giving directions; message subscribing/filtering keeps agents from drowning in irrelevant chatter Masterman+ '24 Empirical.

Codifying coordination — the 2023 frameworks

Three foundational frameworks each codify context in a different way, and their ablations are the earliest hard evidence that the codification matters more than the model.

ChatDev

Roles + chat chain

Role-played agents (CEO/CTO/programmer/reviewer/tester) instantiated by symmetrical "inception prompts," sequenced by a "chat chain." Removing roles from the system prompts is the single largest performance drop. Qian+ '23

MetaGPT

Encoded SOPs + structured artifacts

Human Standardised Operating Procedures compiled into prompt sequences; agents communicate via documents and diagrams (PRD, design, interfaces) over a publish-subscribe pool, not free dialogue. 85.9% Pass@1 HumanEval. Hong+ '23

AutoGen

Conversable agents + conversation programming

Agents (LLM/human/tool-backed) whose interaction is "programmed" by fusing natural language and code; control flow emerges from auto-reply with "no extra control plane." The framework-primitive end. Wu+ '23

▦ The throughline

MetaGPT's central argument — that structure (typed artifacts, role-scoped subscriptions), not more conversation, is what reduces cascading hallucination — is the 2023 ancestor of the 2026 "codified context" thesis. Both say: write the coordination and the role knowledge down in a structured form the agent must consume. Hong+ '23 Vasilopoulos '26

The scaffolding lineage

The benchmark that anchors all coding-agent work, SWE-bench, frames the core problem as one of context: 2,294 real GitHub tasks over codebases averaging 438k lines, where "solving SWE-bench requires identifying the relatively small number of lines that need to be edited… amongst a sea of context," and performance "drops considerably" as context length grows Jimenez+ '23 Empirical.

SWE-agent answered with the Agent-Computer Interface: treat the LM as "a new category of end user" and design its command set, feedback format, and history management deliberately. Its ACI collapses old observations, suppresses verbose search output, and runs a linter on edits — and lifts resolution from 3.8% to 12.47% with the model held fixed Yang+ '24 Empirical.

careful ACI design can substantially improve LM agent performance without modifying the underlying LM's weights… all content has a fixed cost in memory and computation for LMs and distracting context can harm performance.— SWE-agent Yang+ '24

Confucius takes the lineage to its strongest claim — that the scaffold can outrank the model. A weaker model on its scaffold beat a stronger model on a proprietary one (52.7% vs 52.0% on SWE-Bench-Pro), and its gains come from context engineering made first-class: hierarchical markdown working memory, threshold-triggered structured compression, and persistent cross-session notes Wong+ '25 Empirical. Its AX/UX/DX separation (what the agent sees vs what humans see vs what's observable) is a design principle the rest of the corpus implicitly assumes but rarely names.

How agents acquire context — retrieval substrates

Codified context files are one way to get the right information into the window; retrieval is the other. The RAG-for-code survey reframes repository-level retrieval as "a coupled and evolving process involving context construction, retrieval optimization, generation, and environment interaction," not a static retrieve-then-generate pipeline Tao+ '26 Definition. Two parts of its taxonomy matter most here:

  • A repository-as-graph schema — nodes at Directory / Module / Class / Function / Line granularity, edges for Contain / Import / Inherit / Invoke / DataFlow / ControlFlow — a concrete structure for representing a codebase as retrievable context.
  • An autonomy ladder — Level 0 (static one-pass; the non-agentic "Agentless" reaches a competitive 32% on SWE-bench Lite), Level 1 (iterative feedback), Level 2 (planning + tools). The emerging trend is "agentic search": "Claude Code exemplifies this paradigm by replacing static embeddings with dynamic tool usage" (grep/ls/cat in a plan-act-observe loop).

It also marks the design boundary a context layer must decide: long-context models can match RAG on small, well-structured repos, while RAG wins as size/complexity grows Tao+ '26. The code-generation survey complements this by treating the "Context Engine" as a defining axis of deployed tools Dong+ '25:

The same survey catalogs the multi-agent context substrates that prefigure codified context — a blackboard model, von-Neumann instruction/file registers (L2MAC), brain-like tiered memory (Cogito), declarative API memory (CleanAgent) Dong+ '25.

The harness is the architecture

The Claude Code design-space analysis crystallises where the engineering now sits. By a community estimate it cites, only ≈1.6% of the codebase is "AI decision logic"; ≈98.4% is "operational infrastructure" — "minimal scaffolding, maximal operational harness" Liu+ '26 Empirical. The core loop is a single while-loop (ReAct); everything interesting is the harness around it. The paper organises agent coding systems by four design questions:

Design questionClaude Code's answerContrasting answers
Where does reasoning live?model reasons, harness executes/validatesscaffolding-side reasoning (Devin, LangGraph)
How many execution engines?one queryLoop for all surfacesmode-specific engines
Default safety posture?deny-first + layered defense-in-depthcontainer isolation (SWE-agent, OpenHands); git rollback (Aider)
Binding resource constraint?the context windowcompute budget; working memory

Because the context window is the binding constraint, Claude Code runs a five-layer compaction pipeline before each model call (budget reduction → snip → microcompact → context collapse → auto-compact, cheapest first), and offers four extension mechanisms specifically because their differing context costs mean "a single unified API cannot span the full range" Liu+ '26. The separation of reasoning from enforcement is also a security property: a compromised model "cannot override sandboxing or deny-first rules" (see Security).

⟡ My read: two findings that rhyme

"Scaffold can beat model" Wong+ '25 and "98.4% of the system is harness" Liu+ '26 are the same observation from two directions: the durable engineering value in a coding agent is in the harness — context management, tool design, memory, safety — not the model call. Codified context is the declarative, version-controllable slice of that harness, the part a team can author and own without building a runtime. That is, to me, the strongest argument for taking it seriously as infrastructure.

What developers actually assemble

In practice, agent systems are stitched together from multiple frameworks: 96% of top-starred agent projects use two or more, typically an orchestration framework plus a data or multi-agent layer (e.g. LangChain + LlamaIndex, AutoGen + LangChain) Wang+ '25 Empirical. The study warns that popularity ≠ adoption (MetaGPT's tens of thousands of stars vs presence in only 2 studied repos) and that the recurring pains are context-spine problems: memory persistence across turns, message/context de-duplication ("message cooling"), and MCP's prompt overhead (≈11.7k-token prompts, 40% from MCP).

And one rung up, agent-to-agent identity and discovery become their own infrastructure: NANDA's Adaptive Resolver proposes a DNS-analogue that resolves an agent name to a context-tailored endpoint, deliberately separating "Agent Name… from the context in which it is resolved" Zinky+ '25 Proposal — the plumbing beneath MCP/A2A ecosystems, though it is explicitly a speculative, unimplemented design.
