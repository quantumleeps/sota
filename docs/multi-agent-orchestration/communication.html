02 · Communication & Coordination

What crosses the boundary between agents

If frameworks decide who does what, this cluster is about what passes between them — and the corpus's most counter-intuitive finding lives here: across several papers, sending agents more of each other's text does not help and sometimes hurts. The work splits into three questions: how should the channel be standardised, what should actually travel on it, and is the channel itself safe?

Standardising the channel: a taxonomy, not a winner

▦ Five axes for agent communication protocols Definition

The anchor taxonomy classifies nine open-source protocols along five dimensions — counterparty (agent / context / hybrid), payload (structured / conversation / hybrid), interaction state (stateless / session), discovery (static / centralized / decentralized / hybrid), and schema flexibility (single / multiple / evolving) Sander+ '26. Empirically, all seven agent-to-agent protocols carry session state and hybrid payloads; schema flexibility is rising (two protocols negotiate schemas at runtime); decentralized discovery is rare (only one); and privacy, compliance, and policy enforcement are "widely absent" across all nine.

The protocols classified include the names that recur across the rest of the corpus — MCP, A2A, ANP, ACP, agents.json, Agora, LMOS, LAP, agntcy — and the survey's lifecycle paper independently tabulates the same protocol family as the "Communication" leg of its collaboration taxonomy Qi+ '26.

no single protocol is likely to maximize versatility, efficiency, and portability simultaneously. The field will more likely evolve toward a federated, layered protocol stack. — Technical Taxonomy Sander+ '26, Abstract

▦ A negotiation bus, separate from tool-calling Proposal Empirical

LLM-X deliberately reorients from tool/API-centric protocols (it name-checks MCP) to agent-to-agent dialogue, instantiating Contract-Net and FIPA Alternating-Offers over typed JSON envelopes with a policy engine; stricter acceptance policies "improve robustness and fairness but increase latencies and message volume." A 12-hour, 12-agent run sustained 100% round completeness across 9,672 offers at ~6.18 ms mean latency with no drift Lorenzoni+ '26. Caveat carried from the Corpus: the agents are synthetic Python wrappers, so this measures the transport substrate, not LLM negotiation behaviour.

What should travel: less, and at the right time

▦ Controlled disclosure beats sharing transcripts Empirical

DarkForest keeps agents independent and routes only a compact, calibrated belief — "not how many agents support a candidate; it also… which agents support it" — to a coordinator. Exposing full reasoning traces instead of the belief summary lowered GPQA accuracy (36.67% vs 40.00%) while costing more tokens, and the method beats debate/mixture/graph baselines by 2.52–5.00 points at up to 6.5× fewer tokens Y. Li+ '26.

coordination is not equivalent to sharing more text; it is the controlled exposure of compact, policy-permitted, and verifiable evidence. — DarkForest Y. Li+ '26, §Introduction

▦ Refine the transmission unit to the reasoning step Empirical Theory

StreamMA replaces "generate-then-transfer" with a stream that forwards each reasoning step the moment it is produced, pipelining agents over a DAG. Because early steps are more reliable than late ones, starting downstream from the reliable prefix both cuts latency and improves quality — averaging +7.3 pp for Claude over the serial protocol and up to a 26.9× wall-clock speedup, with a closed-form rule (Theorem 1) for when streaming wins per edge Z. Yang+ '26.

the non-uniform quality structure of multi-step LLM reasoning makes when context arrives matter more than how much context arrives. — StreamMA Z. Yang+ '26, Conclusion

⟡ My inference — a single thread under three different mechanisms

DarkForest (disclose a belief, not a transcript), StreamMA (forward a step, not a finished response), and the scaling-law paper on the Debate & Scale page (dense chatter adds no independent evidence) are attacking the same target from three angles: raw inter-agent text is a poor coordination medium. What helps is structuring what crosses the boundary — a calibrated summary, an early reliable prefix, a typed envelope. I read this as the corpus's most actionable communication principle, though no single paper states it at this level of generality. Chips mark the work I reason from.

Before the channel: allocating context into roles

▦ Writing the sub-agent's prompt is a distinct, unsolved skill Empirical

PerspectiveGap moves evaluation one step earlier than task success: can the main agent write each sub-agent a prompt that preserves its information boundary? Across 27 models the average combined pass rate is 14.9% (GPT-5.5 leads at 62.0%); coding-strong Opus 4.7 reaches only 19.1%, and standard interventions — few-shot, scratchpad, more reasoning effort — do not reliably help (medium effort is the sweet spot) Sun+ '26. One failure mode is a security hole: leaking private tests to a coder agent is "a reward-hacking channel created by the orchestration prompt itself."

Generated sub-agent prompts should not be assumed to preserve role-specific information boundaries without inspection. — PerspectiveGap Sun+ '26, Conclusion

Is the channel safe?

⚠ Cheap talk is an adversarial-injection surface

In a 4-player LLM Stag Hunt, a single deterministic deceiver that signals cooperation but defects drives group cooperation to 0% — even though ~50% of honest peers keep cooperating after detecting the betrayal. And merely disclosing a restricted communication topology collapses cooperation (explicit star: 0%) while applying the identical restriction silently preserves it (silent star: 80%), so the failure is meta-reasoning about hidden information, not information loss El Mir+ '26. The authors call cheap talk "structurally equivalent to an adversarial prompt injection attack."

increasing transparency about system structure can decrease system performance when agents engage in meta-reasoning about that structure. — Byzantine Cheap Talk El Mir+ '26, §Topology as attack surface

◇ Open: classical distributed-systems defenses are untested here

The corpus connects LLM coordination to Byzantine fault tolerance and prompt-injection literatures but does not transfer their defenses: verified channels, reputation, and commitment devices are named as open targets El Mir+ '26, and no protocol in the taxonomy ships policy enforcement Sander+ '26.
