CoreCodified Context: Infrastructure for AI Agents in a Complex Codebase
arXiv:2602.20478 · Aristidis Vasilopoulos (Independent Researcher), 2026 · system/tool + experience report · cite: Vasilopoulos '26
An experience report from building a 108K-line C# codebase using Claude Code as the sole code generator over 283 sessions. Argues single-file manifests (CLAUDE.md/AGENTS.md) do not scale, and proposes a three-tier "codified context infrastructure" — an always-loaded constitution (hot memory), 19 invoked domain-specialist agents, and an on-demand knowledge base (cold memory) — totalling 24.2% of the codebase as machine-targeted documentation. Names specification staleness as the primary failure mode. Observational; explicitly claims no causal effect.

CoreEvaluating AGENTS.md: Are Repository-Level Context Files Helpful for Coding Agents?
arXiv:2602.11988 · Thibaud Gloaguen, Niels Mündler, Mark Müller, Veselin Raychev, Martin Vechev (ETH Zurich · LogicStar.ai), 2026 · empirical + benchmark · cite: Gloaguen+ '26
The corpus's strongest skeptic. Controlled evaluation across four agent/model pairs on SWE-bench Lite and a new 138-instance benchmark (AGENTbench) of repos that ship developer-written context files. Finds LLM-generated context files slightly reduce success while adding >20% inference cost; developer files give only marginal gains; instructions are followed (so it is not a compliance failure) but the files largely duplicate existing docs. Recommends omitting auto-generated files and writing only minimal repo-specific requirements.

CoreConfiguration Smells in AGENTS.md Files: Common Mistakes in Configuring Coding Agents
arXiv:2606.15828 · Hélio V. F. dos Santos, Vitor Costa, João E. Montandon, Marco Tulio Valente, Luciana L. Silva (UFMG · IFMG), 2026 · empirical · cite: dos Santos+ '26
Mines the top-100 most-starred repos with AGENTS.md/CLAUDE.md files and derives a catalog of six "configuration smells" — Context Bloat, Skill Leakage, Lint Leakage, Blind References, Init Fossilization, Conflicting Instructions — with detection heuristics. Finds ≥1 smell in 91 of 100 files; Lint Leakage (62%), Context Bloat (42%), and Skill Leakage (35%) are the most common. A hygiene/quality model for the instruction-file artifact.

CoreCode as Agent Harness: Toward Executable, Verifiable, and Stateful Agent Systems
arXiv:2605.18747 · Xuying Ning, Katherine Tieu, Dongqi Fu et al. (UIUC · Meta · Stanford), 2026 · survey · cite: Ning+ '26
The framing survey. Recasts code in agent systems as the harness substrate — executable, inspectable, stateful — and organizes the literature into harness interface, harness mechanisms (planning, memory/context engineering, tool use, plan-execute-verify, harness engineering), and scaling to multi-agent. Names Agentic Harness Engineering as a discipline beyond prompt/context engineering, and argues reliability is bottlenecked by the software layer around the model, with a target property set of executable, inspectable, stateful, governed.

CoreFastContext: Training Efficient Repository Explorer for Coding Agents
arXiv:2606.14066 · Shaoqiu Zhang, Maoquan Wang, Yuling Shi et al. (Microsoft · Shanghai Jiao Tong University), 2026 · system/tool · cite: FastContext '26
Measures that reading and searching consume 46.5% of a coding agent's tokens and 56.2% of its tool-use turns, then delegates exploration to a dedicated trained subagent (4B–30B) that returns a compact <final_answer> block of file-line citations. The solver never sees the explorer's noisy search history. Improves end-to-end resolution up to +5.5% while cutting main-agent tokens up to 60%; a 4B-RL explorer can beat a 30B-SFT one. Argues exploration should be a first-class, trainable interface.

CoreSWE-Explore: Benchmarking How Coding Agents Explore Repositories
arXiv:2606.07297 · Shaoqiu Zhang, Yuhang Wang, Jialiang Liang et al. (SJTU · Xinjiang U · UIUC · CUHK), 2026 · benchmark · cite: SWE-Explore '26
Isolates repository exploration as a standalone, ranked, line-level context-selection task (848 instances, 10 languages, 203 repos), with ground truth derived from ≥2 solution-verified agent trajectories. Finds agentic explorers form a clear tier above classical retrieval; file-level localization is solved but line-level recall is the bottleneck; and context efficiency predicts downstream resolve rate at Pearson r=0.950, with missing core context hurting more than redundant context.

CoreSWE-Pruner: Self-Adaptive Context Pruning for Coding Agents
arXiv:2601.16746 · Yuhang Wang, Yuling Shi, Mo Yang et al. (SJTU LLMSE Lab · SYSU · Douyin), 2026 · system/tool · cite: SWE-Pruner '26
A 0.6B "neural skimmer" inserted as middleware at the agent–environment boundary, pruning file-read observations at line-level granularity given an agent-emitted natural-language "Goal Hint." Reports read operations consume 76.1% of agent tokens, and cuts tokens 23–54% while preserving or improving success and code syntax (AST correctness 87.3% vs 0.29% for token-level pruning). Positioned as orthogonal to history-compression methods — it prunes the environment's observations, not the trajectory.

CorePeek: Context Map as an Orientation Cache for Long-Context LLM Agents
arXiv:2605.19932 · Zhuohan Gu, Qizheng Zhang, Omar Khattab, Samuel Madden (MIT CSAIL · Stanford), 2026 · system/tool · cite: Peek '26
Maintains a small, constant-sized "context map" inside the prompt that caches reusable orientation knowledge about a recurring external context (corpus or repo), built and maintained by a Distiller→Cartographer→Evictor cache policy with per-item IDs and a hard token budget. Evaluated on a production coding agent (OpenAI Codex) among others: improves quality 6–34% over the base agent while persisting full history hurts (inflating tokens ~9–10× for negative gains). Deliberately discovered, not hand-crafted.

CoreOn Problems of Implicit Context Compression for Software Engineering Agents
arXiv:2605.11051 · Kirill Gelvan, Igor Slinko, Felix Steinbauer et al. (JetBrains Research · TU Munich), 2026 · empirical (negative result) · cite: Gelvan+ '26
A cautionary negative result. Tests an In-Context Autoencoder that compresses agent context into continuous "memory tokens." It works for single-shot QA and code-understanding but fails on multi-step SWE-bench Verified — solving 7 issues vs the uncompressed baseline's 19 (p=0.013) — despite extending the effective window and enabling 40% longer trajectories. Diagnoses two causes: reconstruction errors that compound over steps (hallucinated paths break exact-match edit tools), and a training objective with no incentive to preserve information for future steps.

CorePROJECTMEM: A Local-First, Event-Sourced Memory and Judgment Layer for AI Coding Agents
arXiv:2606.12329 · Ripon Chandra Malo, Tong Qiu (University of Utah), June 2026 · system/tool · cite: PROJECTMEM '26
Records development as an append-only plain-text event log, deterministically projects it into AI-readable summaries served over MCP, and adds a deterministic pre-action judgment gate ("you tried this 2 days ago — it failed") keyed to the project's own logged failures. Coins Memory-as-Governance: memory that acts on the next action rather than only answering. Ships secret-redaction-by-default and treats the immutable log as an audit/provenance trail. Evaluated via a 207-event, 10-project self-study; no causal claim.

CoreSkillWiki: A Living Knowledge Infrastructure for Agent Skills
arXiv:2606.16523 · Dingcheng Huang, Yuda Ding, Bingshuo Liu et al. (Harbin Institute of Technology · Tencent · NTU), 2026 · system/tool (demo) · cite: SkillWiki '26
Treats agent skills as versioned, governed "capability assets" rather than ad-hoc prompts — "Wikipedia for knowledge, GitHub for code, and this for skills." A Skill Provenance Graph links knowledge sources → skills → executions → validations → versions; an 8-state lifecycle and Git-style diffs/reviews/releases make evolution traceable, reviewable, and reversible. A demo with one quantitative result (99/125 heterogeneous artifacts converted into governed skill candidates) and no baseline.

CoreDive into Claude Code: The Design Space of Today's and Future AI Agent Systems
arXiv:2604.14228 · Jiacheng Liu, Xiaohan Zhao, Xinyi Shang, Zhiqiang Shen (MBZUAI · UCL), 2026 · system analysis · cite: Liu+ '26
A source-grounded reverse-engineering of Claude Code (TypeScript v2.1.88, ~512K lines). Maps a thin reactive agent loop (≈1.6% of the code) wrapped in a dense deterministic harness (≈98.4%), traced through a four-level CLAUDE.md hierarchy, a five-layer context-compaction pipeline, context-cost-ordered extensions (hooks/skills/plugins/MCP), deny-first permissions, and summary-only subagent isolation. Argues the surrounding harness — not decision scaffolding — is the differentiator as models converge, and stresses the guidance (probabilistic) vs enforcement (deterministic) split.

SupportRepoMirage: Probing Repository Context Reasoning in Code Agents with Perturbations
arXiv:2605.26177 · Hanyu Li, Yichi Zhang, Speed Zhu et al. (BUPT · Tsinghua · Tencent), 2026 · benchmark + empirical · cite: RepoMirage '26
Uses semantics-preserving perturbations (dependency-path indirection, runtime-target masking, value externalization) as a diagnostic on SWE-bench Verified. Shows strong benchmark scores do not reflect true cross-file context reasoning: resolved rate drops 66.8%→25.3% on perturbation-derived tasks, with agents exhibiting "exploration drift" — accessing more files without turning them into structure. A structure-first scaffold (RepoAnchor) that writes an intermediate INSTRUCTION.md recovers much of the loss.

SupportEvolving Agents in the Dark: Retrospective Harness Optimization via Self-Preference
arXiv:2606.05922 · Wenbo Pan, Shujie Liu, Chin-Yew Lin et al. (City University of Hong Kong · Microsoft Research Asia), 2026 · system/tool + empirical · cite: Pan+ '26
Optimizes an agent's full harness — instructions, skills, and executable tools on disk — from its own past unlabeled trajectories in a single offline pass, using self-preference (self-validation + self-consistency) instead of ground-truth grading. Raises SWE-Bench Pro pass rate 59%→78% with no external grading, beating memory-only baselines and matching a validation-feedback optimizer at ~3× less compute. Shows an agent can author its own context artifacts (READMEs, checklists, verification tools) from experience.

SupportMemDocAgent: Memory-Guided Long-Horizon Agent for Repository-Level Code Documentation
arXiv:2605.14563 · Suyoung Bae, Jaehoon Lee, Changkyu Choi et al. (Sungkyunkwan University · University of Oslo), 2026 · system/tool · cite: MemDocAgent '26
Reframes repository documentation as a long-horizon agentic task: a single agent traverses the whole repo in one trajectory with a persistent shared store (RepoMemory) plus dependency-aware traversal and NLI cross-document conflict verification. Eliminates the ~50% repeated-retrieval overlap and 13% cross-document inconsistency of memoryless pipelines (inconsistency −75.5%, read-time −41%). Relevant as a project-memory + codified-context-generation mechanism — documentation as a provisioning artifact for downstream agents.

SupportOrganizational Control Layer: Governance Infrastructure at the Execution Boundary of LLM Agent Systems
arXiv:2606.04306 · Tianyu Shi, Yang Mo, Yiou Liu et al. (McGill · Purdue · UNSW · UCLA · NYU · Aimaikj Research), 2026 · system/tool + empirical · cite: Shi+ '26
A model-agnostic governance layer that intercepts agent-generated actions before execution and either approves, revises, blocks, or escalates them against policy, emitting an audit trace. Demonstrated on adversarial buyer–seller negotiation (not coding): cuts unsafe executions 88%→0% and raises valid (compliant) success 12%→96%. Included for its execution-boundary pattern and audit/escalation vocabulary, which transfer to tool-using coding agents; the domain transfer is the caveat.

SupportRegulating the Machine Contributor: Governance and Policy Alignment in Open Source
arXiv:2606.14594 · Jassem Manita, Aziz Amari (University of Tunis El Manar · University of Carthage), 2026 · empirical policy analysis · cite: Manita & Amari '26
Compares six open-source projects' AI-contribution policies across a six-dimensional taxonomy (disclosure, responsibility, oversight, licensing, enforcement, maintainer workload) and maps them to formal frameworks (EU AI Act, NIST AI RMF, ISO 42001). Its signal claim: "AI lowers the cost of producing contributions but not the cost of trustworthy human review." Surfaces provenance trailers (Generated-By:/Assisted-By:) as codified-context artifacts and names maintainer review-capacity as the universally ungoverned gap.
