A reference model, and the bets this corpus suggests

⟡ This entire page is my inference

Everything below is my reasoning across this corpus of 17 papers, not a claim any single paper makes and not a forecast of where the field is going. Citation chips mark what I reason from, not sources that endorse the conclusion. The corpus is a June-2026, search-shaped sample of preprints; a different sample could support different bets. Read this as a structured opinion, scoped to these papers.

A reference model: the coding agent's context stack

The 17 papers don't share a vocabulary, but they slot cleanly into five layers wrapped by a sixth. I find this stack the most useful way to hold the corpus in one frame — each layer is where one cluster of papers does its work.

Layer 1 · durable spine

The curated instruction file

A minimal, human-authored, version-controlled AGENTS.md/CLAUDE.md — kept lean, with depth pushed to on-demand reference. Gloaguen+ '26 dos Santos+ '26 Vasilopoulos '26

Layer 2 · provisioning

Just-in-time context objects

Machine-generated, task-scoped artifacts — explorer evidence blocks, an INSTRUCTION.md, pruned observations — minted per task and discarded. FastContext '26 RepoMirage '26 SWE-Explore '26

Layer 3 · working-context manager

Fidelity-preserving reduction

Selective pruning and orientation caching that keeps the decision-relevant subset exact — never lossy re-encoding of what a tool will match. SWE-Pruner '26 Peek '26 Gelvan+ '26

Layer 4 · persistent memory

Legible cross-session state

Append-only, provenance-linked project memory and knowledge bases — auditable, not vector-fuzzy. PROJECTMEM '26 SkillWiki '26 MemDocAgent '26

Layer 5 · governance boundary

Deterministic enforcement, outside the model

Approve/revise/block/escalate at the execution and contribution boundaries, with an audit trace. Shi+ '26 Liu+ '26 Manita & Amari '26

Wrapper · the substrate

The harness, increasingly self-modifying

All five layers are harness state — executable, inspectable, stateful, and now rewritten by the agent itself. Ning+ '26 Pan+ '26

The bets this corpus suggests

Bet 1 · Keep the always-loaded spine minimal; earn every loaded token

The strongest skeptical evidence says presence is not value: auto-generated files add cost and reduce success Gloaguen+ '26, popular files are bloated and leaky dos Santos+ '26, and the working systems push the bulk of context to on-demand tiers Vasilopoulos '26 Liu+ '26. The bet: a small, human-curated spine plus deep on-demand reference beats a fat always-loaded file.

Bet 2 · Decouple finding context from using it

Independent results converge on separating exploration from solving and passing a structured artifact across the seam FastContext '26 RepoMirage '26, with context quality predicting success SWE-Explore '26. The bet: a clean context interface between a finder and a solver is worth more than a bigger single agent.

Bet 3 · Optimize for fidelity of the relevant subset, not compression ratio

Compression ratio doesn't track performance Gelvan+ '26; map presence beats map size Peek '26; line-level beats token-level SWE-Pruner '26; missing context beats redundant context as a danger SWE-Explore '26. The bet: keep the load-bearing tokens exact and drop the rest — a coding agent is a brittle exact-match machine downstream of its context.

Bet 4 · Prefer legible, auditable persistence over vector recall

For coding specifically, the corpus chooses append-only logs, provenance graphs, and deterministic projections over embedding stores PROJECTMEM '26 SkillWiki '26. The bet: memory you can diff, review, and trust outperforms memory that is merely similar — and naturally doubles as provenance.

Bet 5 · Put enforcement at the boundary, deterministic and outside the model

The same instinct recurs as a control layer Shi+ '26, a deny-first permission system Liu+ '26, and a named harness property Ning+ '26. The bet: guidance can be probabilistic, but enforcement should be deterministic and live where the agent acts — not inside the weights.

Bet 6 · Treat context and harness as versioned artifacts you can roll back

As context becomes self-maintaining Pan+ '26 SkillWiki '26, the determinism and provenance of the persistence layer PROJECTMEM '26 become the safety mechanism. The bet: change contracts, audit trails, and rollback for the context spine matter more the more the agent edits it.

◇ Where I'd hold back — bets I would not place on this corpus

  • "Auto-generated context files are useless." Too strong. The evidence is single-sample and Python-only, and the files do help in sparsely-documented repos Gloaguen+ '26. The defensible claim is about curation, not abolition.
  • "Self-optimizing harnesses are production-ready." One round, one backbone, self-graded, with an acknowledged entrenchment risk Pan+ '26. I would not run autonomous harness edits without regression gates.
  • "An execution-boundary layer solves governance." The strong numbers are from e-commerce, not coding Shi+ '26; the coding-adjacent study says trustworthy review at machine speed is unsolved Manita & Amari '26.
  • Any single architecture winning. The corpus supports the principles above, not a verdict between a subagent, a context map, or an event log. I would bet on the pattern, not the implementation.
