The 10 papers

Every citation chip on this site resolves to an entry below, and each entry links onward to the paper on arXiv — the chain is always claim → chip → bibliography → arXiv. This is a review of these 10 papers and nothing else: all are arXiv preprints posted in the single week of 15–22 June 2026, surfaced by an automated relevance search across ~14 agent subfields and curated to 10 for topical spread. Most are un-peer-reviewed preprints; several are benchmarks or single-model studies. The per-entry notes and type tags say which.

⟡ How this corpus was assembled — read this first

These ten papers were drawn from a pool of 319 candidates fetched on 22 June 2026 by an automated multi-source search (arXiv + OpenAlex + Hugging Face) spanning fourteen agent subfields — memory, multi-agent orchestration, tool use, planning, RAG, evaluation, RL, code agents, computer use, long context, and more. The pool was filtered to the 240 papers genuinely posted 15–22 June 2026 (verified by arXiv id), embedded and relevance-ranked, then hand-curated down to one representative per subfield to make a deliberately broad cross-section. It is therefore a search-shaped, one-week sample chosen for breadth — not a census of the week, and emphatically not a verdict on the field. Where this site says "across these papers," it means exactly these ten. Strong work that lacked a fetchable arXiv id (e.g. an OpenAlex-only multi-agent-orchestration survey) was excluded.

Cognition — what agents know, remember, and retrieve

CoreAdaMem: Learning What to Remember for Personalized Long-Horizon LLM Agents
arXiv:2606.21144 · Xingyu Chen, Rui Wang, Zhaopeng Tu, Liefeng Bo (Shanghai Jiao Tong Univ.; Tencent), 2026 · empirical + benchmark · cite: Chen+ '26
Reframes long-term agent memory as a write-control problem: instead of "remembering everything," an agent should learn a role-specific Memory Policy — refined from weekly QA feedback with failure-rollback — that keeps only what each user actually cares about. Reports up to +9.0% QA accuracy over a uniform-extraction baseline (Mem0) while shrinking memory volume 9.1%, and diagnoses that the bottleneck is inferring preferences from weak/implicit feedback, not following them. Contributes AdaMem-Bench (golden-memory annotations) and the Memory Efficiency Ratio (MER).

CoreUncertainty Decomposition for Clarification Seeking in LLM Agents
arXiv:2606.19559 · Gregory Matsnev (AI Talent Hub, ITMO Univ.), 2026 · empirical + benchmark · cite: Matsnev '26
Argues prompt-based uncertainty is the only practically deployable family for black-box interactive agents, and splits the single confidence scalar into separate action confidence and request uncertainty so an agent has a dedicated channel for goal ambiguity and can seek clarification proactively. Improves clarification F1 by 73% over ReAct+UE (averaged over five 2026 backbones), but documents the bundled costs: a small monotonic "capability dilution" of task success, systematic overconfidence, and a finding that product-aggregated confidence is mostly a trajectory-length proxy. Adds two clarification benchmarks.

CoreCan LLM Agents Infer World Models? Evidence from Agentic Automata Learning
arXiv:2606.16576 · Reef Menaged, Gili Lior, Shauli Ravfogel, Roee Aharoni, Gabriel Stanovsky (Hebrew Univ. of Jerusalem; NYU; Google Research), 2026 · benchmark + empirical · cite: Menaged+ '26
Recasts classic active automata learning as a scalable agent testbed: a tool-calling agent must uncover a hidden deterministic finite automaton (DFA) through membership and equivalence queries, with classic algorithms (L*, TTT) as exact 100%-solving baselines. Finds that frontier reasoning models can do non-trivial interactive discovery but degrade sharply with state count — no model exceeds 25% success at 8–9 states — and issue increasing fractions of non-informative queries as interactions lengthen. A clean instrument for "does the agent build a world model through interaction, or pattern-match?"

CoreDissecting Agentic RAG: A Component Ablation for Multi-Hop QA with a Local 7B Model
arXiv:2606.21553 · Sheroz Shaikh (independent researcher), 2026 · empirical (controlled ablation) · cite: Shaikh '26
A controlled, significance-tested ablation of an agentic-RAG pipeline run entirely on a local Qwen2.5-7B over HotpotQA. The headline: most of the benefit comes from running a short retrieval loop at all, not from adaptivity or depth — fixed hybrid retrieval beats the full rule-based adaptive router by +1.8 EM, and two iterations capture 95% of the gain of five. Surfaces a concrete failure mode (entity-firing heuristics over-route to BM25) and a practical rule: keep reranking, drop decomposition first under latency pressure. Reproducible, no proprietary APIs.

Safety & Robustness — the agent as an attack surface

CoreWhen Lower Privileges Suffice: Investigating Over-Privileged Tool Selection in LLM Agents
arXiv:2606.20023 · Kaiyue Yang, Yuyan Bu, Jingwei Yi, Yuchi Wang et al. (Chinese Academy of Sciences; BAAI; CUHK; Peking Univ.), 2026 · benchmark + empirical · cite: Yang+ '26
Defines and measures over-privileged tool selection — an agent escalating to a higher-privilege tool when a sufficient lower-privilege one exists — via ToolPrivBench (544 scenarios, 8 domains, 5 risk types). Finds it prevalent and model-dependent (six of eleven 2026 models exceed 30% over-privileged-use rate; Qwen3-8B 64.9%, while Claude 4.6 Sonnet / GPT-5.2 / GLM-5 stay under 10%), amplified by transient tool failures, and not fixed by general safety alignment. Proposes a privilege-aware SFT+GRPO recipe that cuts the rate while preserving capability.

CoreLocal LLM Agents as Vulnerable Runtimes: A Source-Code Audit of the Agent Runtime Layer (ClawAudit)
arXiv:2606.21071 · Zhengsong Zhang, Zongze Li, Jiawei Guo, Haipeng Cai (Univ. at Buffalo, SUNY), 2026 · system + benchmark + empirical · cite: Zhang+ '26
Argues a local agent is "not merely a model wrapper… it is a privileged software runtime" — prompt builder, parser, tool dispatcher, skill loader, memory writer, network client, permission gate — and that this implementation layer is an unaudited safety boundary. Builds OpenClawBench (446 real source-level advisories) and ClawAudit, a STRIDE-derived static-analysis ruleset that lifts held-out detection recall from ~14–22% (generic Semgrep/CodeQL) to 67–75%, while honestly reporting low live-code precision (~12%). Names the residual gap: vulnerabilities with no syntactic surface. Note: the paper pseudonymizes products as "OpenClaw", "Nanobot", "ClawHub" — preserved as written.

CoreAutoRAS: Learning Robust Agentic Systems with Primitive Representations
arXiv:2606.21445 · Yang Yue, Xuancheng Zhu, Yuyang Ma, Guoshun Nan et al. (Beijing Univ. of Posts & Telecom.; Fudan; China Telecom), ICML 2026 · framework + empirical · cite: Yue+ '26
Automated design of multi-agent systems that makes robustness a first-class objective rather than a post-hoc patch. Represents a system as a sequence of symbolic "primitives" (jointly encoding topology and behavior, including safety primitives), compiled deterministically, and searches the space with a GFlowNet / trajectory-balance optimizer driven by execution-derived safety signals. Reports the highest average clean accuracy (74.27%) and by far the smallest drop under a four-vector attack suite (2.13% vs −14% to −18% for baselines), with a lightweight 4.29M-parameter learned policy over frozen API LLMs. Scopes its claim to "structural robustness within the evaluated threat model."

Evaluation & Serving — measuring and running agents honestly

CoreBeyond Static Leaderboards: Predictive Validity for the Evaluation of LLM Agents
arXiv:2606.19704 · Dhaval C. Patel et al. (+60 co-authors; IBM AssetOpsBench; Columbia HPML), EMNLP 2026 submission · position + synthesis · cite: Patel+ '26
A meta-evaluation position paper: aggregate-score leaderboards "systematically underspecify the dimensions on which deployed agents are evaluated," and the field should rank configurations by predictive validity (does an in-sample ranking transfer out-of-sample?) over a twelve-tier measurement apparatus. Marshals direct rank-instability evidence — a 149-team competition where public-vs-hidden execution rank correlated ρ=−0.13 (indistinguishable from zero) — and consolidates 14 parallel implementation studies. Explicitly runs no new controlled experiments; its strength is "convergent architectural sensitivity," not independent replication.

CoreLabOSBench: Benchmarking Computer Use Agents for Scientific Instrument Control
arXiv:2606.16802 · Anqi Zou, Han Deng, Chengyu Zhang et al. (Shenzhen Loop Area Inst.; Dalian Univ. of Tech.; CUHK; HKUST), 2026 · benchmark + empirical · cite: Zou+ '26
Pushes computer-use-agent evaluation into a harder regime — feedback-driven control of scientific instruments — with a lightweight, browser-native benchmark of eight instrument simulators (96 subtasks), Playwright-driven, no OS virtualization, scoring intermediate scientific state (e.g. PSNR on focus) not just final completion. Best agent reaches 0.814 average subtask score (human 0.924), but end-to-end workflow success is only 26.3%, exposing severe error accumulation; "GUI grounding is necessary but not sufficient," and "simply adding an agent loop is not enough." Several subtasks score 0.00 for all twelve methods.

CoreSwarmX: Agentic Scheduling for Low-Latency Agentic Systems
arXiv:2606.21401 · Yeqi Huang, Yanwei Ye, Zhan Lu, Luo Mai et al. (Univ. of Edinburgh); Guomin Chen, Bin Gong, Jialian Li et al. (Tencent), 2026 · system + empirical (production) · cite: Huang+ '26
The serving-infrastructure layer of agentic systems: because an agentic workflow's inference time and call structure depend on prompt semantics, conventional cluster schedulers misfire. SwarmX turns routers and scalers into lightweight "scheduler agents" that use compact, distribution-aware neural predictors (e.g. a 35M-param semantic model standing in for an 8B target) to cut tail latency — up to 61.5% P95 reduction on structured pipelines and ~2× production throughput under the same SLO, with sub-millisecond-to-30ms predictor overhead. Treats the cluster scheduler itself as an "agent." Note: evaluated workloads named "OpenClaw" and "Coding Agent" run open Qwen3-family models.
