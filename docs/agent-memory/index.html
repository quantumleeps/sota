Context engineering & agent memory

Through 2025–2026, two conversations merged into one. "Context engineering" — the discipline of curating what goes into a finite context window — and "agent memory" — the persistent store behind it — are converging on a single idea: the context window is the fast, scarce top of a managed memory hierarchy. This site reads sixteen sources on that convergence and tries to separate what's measured from what's hoped.

The state of the art in six claims

These are the positions with enough independent support across these sixteen sources to count as consensus within this corpus — a reading of the papers brought to this review, not a verdict on the field.

1 · The context window is a finite resource, not a container Definition Theory

Performance degrades as tokens accumulate ("context rot"), and models draw on a finite "attention budget" rooted in the transformer's n² cost — a gradient, not a cliff. This premise is shared by practitioners and surveys alike. Anthropic '25 Anthropic '26 (Cookbook) Mei+ '25 → Foundations

2 · Context engineering and agent memory are converging Definition

The surveys now draw a clean line — context engineering manages the window's resources; memory governs persistent cognitive state — and locate their overlap at working memory. In practice they are two views of one tiered store. Hu+ '25 Du '26 Mei+ '25 → Foundations

3 · The practitioner toolkit has consolidated into a few composable primitives Proposal

Compaction, tool-result clearing, external memory/note-taking, just-in-time retrieval, and sub-agent isolation now cover most of what teams need — distinguished by scope, lossiness, and persistence, and chosen by which problem the workload actually has. Anthropic '25 Anthropic '26 (Cookbook) OpenAI '26 → Practice

4 · Memory management is moving from heuristic to learned and self-evolving Proposal Empirical

RL-trained memory operations (AgeMem), evolvable memory skills (MemSkill), evolving context playbooks (ACE) and query-aware budget routing (BudgetMem) are replacing hand-coded schedules — and beating them on quality and cost on long-horizon benchmarks. Yu+ '26 (AgeMem) Zhang+ '26 (MemSkill) Zhang+ '25 (ACE) Zhang+ '26 (BudgetMem) → Memory Systems

5 · The window is being re-imagined as a memory hierarchy Theory

OS abstractions — demand paging, virtual memory, KV-cache garbage collection — are applied directly, with the context window cast as "L1 cache" atop a deeper store. The framing is powerful and the early production data is striking. Mason '26 Santoni '26 (CMV) Kariyappa & Suh '26 → Memory Hierarchy

6 · Memory cuts cost — but doesn't reliably make agents act better Empirical

Purpose-built memory reaches near-full-context quality at a fraction of the cost; yet the one adversarial, decision-focused benchmark finds external memory gives no consistent gain over raw long context, with success decaying as tasks deepen. The cost case is strong; the capability case is not. Chhikara+ '25 (Mem0) He+ '26 (MemoryArena) → Evidence
