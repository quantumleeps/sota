Synthesis

⟡ This whole page is my inference — not the papers' claims

Everything below is my reasoning across these 14 papers. Citation chips mark what I reason from, not sources that agree with my conclusion. Nothing here is a forecast of where the field is going — it is "what this corpus suggests" if you read it as one body of work, scoped to a three-month, search-shaped sample of preprints. Treat it as an argument, not a result.

A reference model that unifies the corpus

No single paper offers a complete memory stack, but the 14 slot together into one. Read as a pipeline, the corpus describes seven layers — and almost every paper is contributing a different layer, which is the strongest evidence that this decomposition is real and not imposed.

Layer 1 · Fast tier

Working memory & context

The context window as a cache: protect KV-cache prefixes, evict by residual utility, maximize information density. TokenPilot '26 GenericAgent '26

Layer 2 · Write gate

Decide what's worth storing

Per-task and per-user gating, learned rather than universal — most of the headroom is here and unrealized. PerMem '26 TaskMem '26

Layer 3 · Store

A structured, evolving substrate

Graph, layered, or patch-versioned — readable and editable; optionally latent for cheapness. FluxMem '26 EvoArena '26 JAMEL '26

Layer 4 · Adjudication

Keep the current state authoritative

Mark stale, resolve implicit conflict, ensure the live belief governs the answer — the corpus's weakest layer. STALE '26 WorldMemArena '26

Layer 5 · Consolidation

Forget on purpose

Distill episodes into skills/SOPs, prune interference, retire stale assets — stop appending. MUSE '26 Bayesian-Agent '26

Layer 6 · Evolution control

Learn to tune the stack

An LLM-driven (or trained) controller that reconfigures retrieval and write policy from failure signal. EvolveMem '26 MemTrain '26

The seventh layer is a verification spine threaded through all of them: update belief only from externally checked outcomes, never from the model's self-report Bayesian-Agent '26. In my reading, that spine is what separates the systems that should be trusted in deployment from the ones that merely score well.

What this corpus suggests — design bets

Bet 1 · Adjudicate at write time, not read time

The single most leveraged result in the corpus is STALE's jump from 8.7% to 68.0% by adjudicating state as memory is written rather than when it is read STALE '26. If retrieval already surfaces the right evidence ~77% of the time and the failure is authority, then the fix belongs upstream of retrieval. I would build the "is this still true?" decision into ingestion.

Bet 2 · Treat updates as non-additive events, and keep memory readable

EvoMem's version-aware patches and FluxMem's editable graph both say: don't silently overwrite, and don't store opaquely EvoArena '26 FluxMem '26. Readable memory is what makes adjudication, forgetting, and audit possible at all — which is why I would treat latent memory as an optimization, not a default.

Bet 3 · Budget memory like a cache; density beats volume

Three independent results — condensed memory matching redundant at a fraction of the tokens GenericAgent '26, cache-aware eviction cutting cost ~87% TokenPilot '26, and latent compression at 2.8–22× JAMEL '26 — point one way: more stored memory is a liability, not an asset, unless it raises decision-relevant density.

Bet 4 · Learn memory from deterministic or verified signals

The learned-memory systems that work end-to-end found a cheap, automatic proxy for "good memory" — masked reconstruction MemTrain '26, code-coverage novelty JAMEL '26, verified task outcomes Bayesian-Agent '26. The ones that stall wait on a signal that isn't cheap (per-user worth PerMem '26). Pick the layer where you have a deterministic signal; hand-engineer the rest for now.

Bet 5 · Evaluate the lifecycle, not recall

WorldMemArena's write→maintain→retrieve→use framing and EvoArena's chain-level evaluation both reveal failures that single-shot recall benchmarks hide WorldMemArena '26 EvoArena '26. If you can only measure one thing, measure whether the agent acts on the current state across a long horizon.

◇ Where I'd hold back — bets I would not place

That any one architecture wins. There is no neutral cross-design comparison in this corpus; every system is its own benchmark's champion. That self-evolution is net-positive by default. Bayesian-Agent and EvoMem both document their own methods hurting in some settings — adaptivity is conditional, not free Bayesian-Agent '26 EvoArena '26. That latent memory is ready for agents that must be audited. Its own proponents call it hard to supervise JAMEL '26. And that the accuracy leaderboards will replicate — with zero significance tests and self-authored benchmarks across the corpus, I treat the diagnostic findings as the durable contribution and the leaderboard wins as provisional.

⟡ If I had to compress this corpus to one sentence

Within these 14 papers, agent memory is being rebuilt from a retrieval problem into a state-management problem — and the systems worth watching are the ones that make memory readable, adjudicate the current state at write time, forget on purpose, and update only from evidence they can verify.
