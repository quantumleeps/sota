Core sources

CoreEvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments
arXiv:2606.13681 · Jundong Xu, Qingchuan Li et al. (NUS · SMU · UW · UCL · UPenn · NTU · MIT), 2026 · benchmark + framework · cite: EvoArena '26
A benchmark for persistent environment evolution (interfaces, code, and preferences that change across versions) plus EvoMem, a "git-like" patch memory that records non-additive updates as an append-only history with pre/post state, rationale, and evidence. Diagnoses the dominant "state-collapse" failure where memory is kept as a single latest snapshot, and shows version-aware memory helps most at chain (multi-step) level.

CoreFluxMem: Rethinking Memory as Continuously Evolving Connectivity
arXiv:2605.28773 · Jizhan Fang et al. (Zhejiang University · Alibaba · MemTensor · Tongji), 2026 · framework + empirical · cite: FluxMem '26
Models memory as a continuously evolving heterogeneous graph (semantic / episodic / procedural layers) whose connectivity is refined online and consolidated offline, recasting context optimization as topological editing of an activated subgraph. Introduces a maturity metric (PEMS) to decide when consolidation should stop. Reports strong results on LoCoMo, Mind2Web, and GAIA.
arXiv abstract ↗
CoreTokenPilot: Cache-Efficient Context Management for LLM Agents
arXiv:2606.17016 · Buqiang Xu et al. (Zhejiang University · UESTC · Xidian · HomologyAI), 2026 · system/tool · cite: TokenPilot '26
Argues that text-pruning and memory-eviction methods save tokens but mutate the prompt layout, shattering KV-cache prefix continuity and erasing the savings. Reconciles text sparsity with cache alignment via global ingestion-aware compaction (stable prefixes) and a local active→completed→evictable lifecycle gated by residual utility. Frames working memory as a hardware-cache problem, not just a token-count problem.
arXiv abstract ↗
CoreWorldMemArena: Evaluating Multimodal Agent Memory Through Action–World Interaction
arXiv:2605.29341 · Chengzhi Liu, Yuzhe Yang et al. (UC Santa Barbara · JPMorganChase · ETH Zurich · Stanford · JHU · CMU), 2026 · benchmark · cite: WorldMemArena '26
Reframes multimodal memory evaluation as a four-stage lifecycle — write → maintain → retrieve → use — with 461 multi-session tasks and stage-level annotations that localize where memory fails. Head-to-head across long-context, hand-designed memory systems, and self-managing harness agents. Headline finding: better writing/storage does not guarantee better answers; using the right evidence at answer time is the binding constraint.
arXiv abstract ↗
CoreEvolveMem: Self-Evolving Memory Architecture via AutoResearch for LLM Agents
arXiv:2605.13941 · Jiaqi Liu, Xinyu Ye et al. (UNC-Chapel Hill · UC Berkeley · UC Santa Cruz), 2026 · system/tool · cite: EvolveMem '26
Observes that most memory systems evolve stored content but freeze the retrieval infrastructure at deployment. Exposes the full retrieval configuration as an action space and lets an LLM-driven diagnosis module self-tune it in a closed "AutoResearch" loop — reading per-question failure logs, proposing changes (even parameters absent from the original space), and reverting regressions. Reports positive cross-benchmark transfer.
arXiv abstract ↗
CorePersonalize-then-Store: Benchmarking and Learning Personalized Memory for Long-horizon Agents
arXiv:2605.25535 · Yeonjun In et al. (KAIST), 2026 · benchmark + empirical · cite: PerMem '26
Argues that "worth-storing" is user-specific, not domain-intrinsic, and introduces PerMem-Bench (multi-year, multi-domain histories for 20 users) to evaluate personalized memory. Proposes lightweight session-level storage gating. Finds large retention gains under perfect (oracle) gating but that current gating methods can't realize them — a precise measure of unrealized headroom.
arXiv abstract ↗
CoreMemTrain: Self-Supervised Context Memory Training
arXiv:2606.03197 · Ziheng Li, Xingrun Xing et al. (Peking University · Samsung Research Beijing), 2026 · empirical (training framework) · cite: MemTrain '26
Trains a model's fixed-size context memory with two self-supervised proxy tasks over unlabeled Wikipedia — end-to-end masked reconstruction (outcome-level) and intermediate memory recall (process-level) — as a better initialization for downstream memory post-training. Shows strong length generalization (7k→896k tokens) and that memory training raises a ceiling post-training scaling alone cannot reach.
arXiv abstract ↗
CoreGenericAgent: A Token-Efficient Self-Evolving LLM Agent via Contextual Information Density Maximization
arXiv:2604.17091 · Jiaqing Liang, Jinyi Han et al. (Fudan University · Shenzhen Aquaintelling / A3 Lab), 2026 · system/tool · cite: GenericAgent '26
A general agent built on one principle: maximize decision-relevant information per unit of finite context budget (target <30k tokens). Combines a minimal atomic toolset, an on-demand four-layer memory (index / fact / SOP / raw archive), episodic→procedural→code consolidation under a "No Execution, No Memory" rule, and aggressive context trimming. Argues a non-retrieval, organization-based memory can beat embedding/vector-DB systems on long-term recall.
arXiv abstract ↗
CoreSTALE: Can LLM Agents Know When Their Memories Are No Longer Valid?
arXiv:2605.06527 · Hanxiang Chao, Yihan Bai et al. (Wuhan University · CUHK · HKUST), 2026 · benchmark (+ prototype) · cite: STALE '26
Isolates implicit conflict — when a later observation invalidates an earlier memory without explicit negation — across 400 expert-validated scenarios and three probes (state resolution, premise resistance, implicit policy adaptation). Names the "current-state adjudication gap": updated evidence is retrieved but does not govern the answer. A prototype (CUPMem) that adjudicates at write time lifts a weak backbone from 8.7% to 68.0%.

Supporting sources

SupportTaskMem: Task-Focused Memorization for Multimodal Agents
arXiv:2605.31075 · Tao Zou, Yichen He et al. (ByteDance Seed · Fudan University), 2026 · empirical (+ framework) · cite: TaskMem '26
Frames memory generation as a learnable RL policy: first trained for general memory quality, then adapted online via a tiny (2,048-parameter) steering vector so the agent learns what to remember for its task. Demonstrated on streaming multimodal (video) VQA; positions the adapter as a form of "parametric personalized memory" that resists catastrophic forgetting.
arXiv abstract ↗
SupportJAMEL: Joint Agent Memory and Exploration Learning via Novelty Signals
arXiv:2606.01528 · Shizuo Tian et al. (Tsinghua · SYSU · Baidu · Tongji · Peking University), 2026 · empirical (system) · cite: JAMEL '26
Trains a compact latent memory jointly with exploration, using a deterministic, annotation-free novelty signal (GUI code coverage) as step-level supervision. Each interaction is compressed into a single latent memory token (a frozen-VLM hidden state) and retained as a soft prefix — matching cloud models' exploration depth at 2.8×–22× lower token cost. A worked case for the "latent memory is efficient but hard to supervise" tension.
arXiv abstract ↗
SupportMUSE-Autoskill: Self-Evolving Agents via Skill Creation, Memory, Management, and Evaluation
arXiv:2605.27366 · Huawei Lin et al. (ByteDance · Rochester Institute of Technology), 2026 · system/tool + framework · cite: MUSE '26
Treats skills as long-lived, testable assets managed through a five-stage lifecycle, with a multi-level memory: short-term (compressed task context), long-term (uncompressed cross-session notes), and a per-skill .memory.md experience store. A deliberate design stance for the thread: transferable capability (the skill) is shipped, but accumulated experience (the memory) is kept per-agent.
arXiv abstract ↗
SupportMemDreamer: Decoupling Perception and Reasoning for Long Video Understanding via Hierarchical Graph Memory and Agentic Retrieval
arXiv:2606.07512 · Cong Chen, Guo Gan, Kaixiang Ji et al. (Ant Group · Zhejiang University · Central South University · HKUST-GZ), 2026 · system/tool + empirical · cite: MemDreamer '26
Decouples perception from reasoning: a perception model streams long video into a persistent, purely-textual three-tier graph memory; a separate reasoning model answers by agentic tool-augmented retrieval over that memory (~2% of full-context tokens). Evidence that structured graph memory plus multi-hop traversal beats flat embedding RAG and naive full-context dumping — and that good memory organization lets reasoning ability express itself.
arXiv abstract ↗
SupportBayesian-Agent: Posterior-Guided Skill Evolution for LLM Agent Harnesses
arXiv:2606.08348 · Xiaojun Wu, Cehao Yang et al. (IDEA Research · HKUST-GZ · DataArcTech), 2026 · framework + empirical · cite: Bayesian-Agent '26
Treats reusable harness assets (skills, SOPs, failure patches) as evidence-bearing Bayesian hypotheses about whether a frozen model will succeed, updating belief only from externally verified outcomes (never the model's self-assessment). A posterior maps to five inspectable actions — explore, patch, split, compress, retire — making consolidation/forgetting auditable. Shows online skill-memory growth can hurt as well as help.

◇ Scope & provenance note

These 14 papers were surfaced by an automated arXiv-relevance ranking for the query "LLM agent memory," then filtered to a January 2026 date floor. The selection is therefore shaped by query phrasing and by what was on arXiv in mid-2026 — it over-represents Chinese-lab and large-industry preprints and the May–June 2026 window, and almost all sources are un-peer-reviewed preprints with no independent replication. Model and tool names (e.g. GPT-5.5, Claude Opus 4.6, Gemini 3.1, DeepSeek V4) are reproduced exactly as written in the sources. Read the site as a faithful reading of this sample, not a verdict on the field.
