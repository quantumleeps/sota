Page 05 · what the corpus itself flags

Limitations & open problems

These are the gaps the papers admit to — plus the ones visible only when you read all 14 together. I have grouped them into families. Several are not weaknesses of individual systems but of the corpus as a body of evidence.

1 · The evaluation base is fragile

This is the limitation that colours everything else. The corpus advances faster than its ability to measure itself.

⚠ Self-evaluation, synthetic data, no significance

Most "beats baseline" numbers come from a benchmark the proposing paper introduces, scored by an LLM judge, with no significance test and no independent replication. Concretely: PerMem-Bench is fully synthetic (20 users) PerMem '26; EvolveMem's MemBench evaluation is 28 samples with default hyperparameters tuned on just 2 LoCoMo samples EvolveMem '26; MUSE covers 51 of 94 tasks and its authors note "confidence intervals for individual tasks are wide" MUSE '26; STALE's scenarios are LLM-generated then human-validated STALE '26; and at least two papers (EvoArena, MemDreamer) contain internal numeric inconsistencies between main text and appendix/tables EvoArena '26 MemDreamer '26.

2 · Adjudication — knowing which memory is current — is unsolved

The corpus's central open problem (see Consistency) remains open even in the papers that tackle it. STALE's own fix depends on a predefined state schema and concedes that schema-free, open-domain state inference is "fundamental and far from solved." STALE '26 Empirical EvoMem's traceable history can itself introduce ambiguity about which state is current on hard preference conflicts. EvoArena '26 Empirical And WorldMemArena finds the "maintain" stage — update plus distractor rejection — is where most systems fail outright. WorldMemArena '26 Empirical

3 · Evolving memory has a cost the papers rarely measure

◇ The overhead of "self-evolving"

Adaptive memory runs extra LLM calls, and most papers don't price them. FluxMem's online refinement and offline consolidation use iterative LLM calls whose latency, API cost, and token consumption are explicitly not measured FluxMem '26; a full EvolveMem self-evolution run takes 25–35 minutes on a single sample EvolveMem '26; MemTrain adds a self-supervised pre-training stage with real GPU cost MemTrain '26. The efficiency-focused systems (TokenPilot, GenericAgent) are the exception that proves the rule — they measure cost because cost is their thesis.

4 · Learned memory inherits a supervision problem

Moving memory from heuristics to learning relocates the difficulty to the training signal (see Learning), and the corpus shows the seams:

  • JAMEL's novelty signal is GUI-specific (code coverage); embodied analogues are mentioned but not evaluated, and dense interfaces still cause "entrapment." JAMEL '26
  • TaskMem needs a "richness reward" to stop the policy reward-hacking toward empty memories, relies on as few as 5–10 feedback items, and uses the same model (GPT-4o) to both build preferences and answer. TaskMem '26
  • Bayesian-Agent documents that online skill-memory growth is non-monotonic — it can make a frozen model worse when evidence is sparse. Bayesian-Agent '26
  • No paper demonstrates a learned-memory recipe transferring across domains without a domain-specific signal.

5 · Narrow scope and modality

Most systems are demonstrated in one regime, and generalization is conjecture. TaskMem covers only episodic memory (semantic/visual memory left to future work) TaskMem '26; EvoArena spans three evolution regimes and calls for robotics/embodied/scientific extensions EvoArena '26; PerMem gates storage only and offers no mechanism to evict a wrongly-stored entry PerMem '26; GenericAgent's skill-tree maintenance (merging, deprecating) is entirely manual GenericAgent '26; and multimodal memory remains largely text-centric, with visual detail compressed into captions and lost WorldMemArena '26 MemDreamer '26.

6 · Latent memory is opaque where the field needs auditability

The cheapest memory in the corpus is the hardest to inspect. JAMEL's own framing — latent vectors "lack human-readable semantics" and are hard to supervise JAMEL '26 — sits in direct tension with the consistency work, which needs memory you can read, mark stale, and adjudicate. The corpus offers no way to debug a latent memory that has silently encoded a wrong or outdated belief.

⟡ The cross-cutting tension I see

The corpus wants two things that pull apart. It wants memory to evolve — grow richer, restructure, accumulate skills and experience — and it wants memory to stay stable and correct — never serve a stale belief, never accumulate clutter, always know the current state. Every paper sits somewhere on that adaptivity-vs-stability axis, and none resolves it: the self-evolving systems (FluxMem, EvolveMem, EvoMem) are the ones most exposed to the staleness and ambiguity problems the consistency papers (STALE, WorldMemArena) measure. A second, quieter tension runs alongside: autonomy vs auditability — the more memory self-evolves (and the more it goes latent), the less a human or a verifier can inspect what it has come to believe. Bayesian-Agent's insistence on updating only from externally verified evidence reads, to me, as an early answer to that second tension. I think the field's progress will be measured by how well the next systems hold both ends at once, not by another point on a recall leaderboard.
