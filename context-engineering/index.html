State of the Art · mid-2026

Context engineering for AI coding agents

As coding agents moved from autocomplete to autonomous multi-step work over real codebases, the dominant lever on their behaviour stopped being the model and the single prompt. It became the context supplied to them — and the frontier practice is to codify that context as durable, version-controlled, machine-addressed artifacts: AGENTS.md/CLAUDE.md, Cursor rules, skills, MCP tools, sub-agent specs, persistent notes.

This is a synthesis of a 33-document research corpus — one anchor paper, five core and four support papers, and 23 references harvested from the anchor — spanning empirical studies of context files, context-engineering theory, multi-agent frameworks, coding-agent benchmarks and scaffolds, security, and developer practice. It reads as a connected set of pages; each idea is tied to its source.

The state of the art in five claims

Across the corpus, five claims recur with enough independent support to count as the field's working consensus.

1 · Context — not raw model capability — is the dominant near-term lever

A purpose-built agent–computer interface lifted SWE-bench resolution from 3.8% to 12.47% without changing the model Yang+ '24. A scaffold can make a weaker model beat a stronger one: Confucius + Claude 4.5 Sonnet (52.7%) edged Claude 4.5 Opus on a proprietary scaffold (52.0%) on SWE-Bench-Pro Wong+ '25. The MAST study found multi-agent failure is "not merely a function of the underlying model" — better design alone added +9.4% with the same model Cemri+ '25. Empirical

2 · Codified context measurably improves efficiency and correctness

The first controlled isolation of a single context artifact — a paired within-task design over 124 real PRs — found AGENTS.md cut median runtime 28.64% and median output tokens 16.58% at comparable completion Lulla+ '26. A layered context pipeline doubled single-shot success (80% vs 40%) over a CLAUDE.md baseline Haseeb '25. Evolving "playbook" contexts added +10.6% (agents) / +8.6% (finance) Zhang+ '26; codified expert knowledge lifted output quality 206% in an industrial study Ulan uulu+ '26. Empirical → see the full evidence ledger

3 · Context files are now a real software-artifact class — with anatomy, evolution, and debt

Multiple independent studies converge on a shared anatomy: a shallow heading hierarchy, action-oriented content, and a striking neglect of non-functional requirements Chatlatanagulchai+ '25a Santos+ '25 Mohsenimofidi+ '26. The largest study (2,303 files) shows they "evolve like configuration code" and coins context debt for their low readability and length Chatlatanagulchai+ '25b. Empirical → the anatomy of a context file

4 · The single flat file does not scale — the frontier is structured, tiered, retrievable infrastructure

The anchor's thesis is exactly this: "a 1,000-line prototype can be fully described in a single prompt, but a 100,000-line system cannot," so it replaces the manifest with hot/cold memory tiers, routed sub-agents, and an MCP knowledge base Vasilopoulos '26. Parallel moves: ACE's incremental "playbooks" Zhang+ '26, Confucius's hierarchical memory + cross-session notes Wong+ '25, and the survey formalism of context as an assembled component set Mei+ '25. Proposal Theory

5 · Codification cuts both ways — it adds new failure and attack surface

The same artifacts that steer agents can misfire: the planner→coder handoff loses 75.3% of failures' worth of detail Lyu+ '26; contexts "collapse" and degrade Zhang+ '26; agents "trust documentation absolutely," so stale specs fail silently Vasilopoulos '26. And context files are an injection/supply-chain surface that needs system-level monitoring Zhang+ '26. Empirical Theory → failure modes · security
