State of the Art · Spec-Driven Development & the AI SDLC

When code is cheap, the specification becomes the program

Across these 14 papers — all arXiv preprints from April–June 2026 — the recurring move is to push engineering's centre of gravity upstream of code. As coding agents make implementation abundant, the scarce, value-bearing artifact becomes the specification: the precise statement of intent, constraints, and acceptance evidence that an agent executes against. The papers converge less on "which model writes better code" and more on the process question — which discipline turns natural-language intent into verifiable engineering output, at a throughput humans can no longer manually inspect.

It is a reading of this corpus — a search-shaped, three-month sample of mostly un-peer-reviewed preprints, several of them position or vision papers — not a complete survey and not a claim about where the field is heading. What recurs across these 14 papers

Claims with enough independent support within this corpus to count as a shared view here — not field-level verdicts.

1 · The specification/intent is the primary artifact; code is the cheap output Theory Definition

Multiple papers independently argue the locus of engineering work moves from authoring code to authoring intent. Marron makes specs "the primary artifact… from which flows implementation, testing, deployment, and interaction with AI agents" Marron '26; De La Cruz names this the shift from "code-centric" to "intent-centric" engineering De La Cruz '26; Alenezi puts "intent specification" as the least AI-substitutable competency Alenezi '26; Mandl observes "the amount of generated code is not a meaningful success measure" Mandl+ '26. → The intent-centric turn

2 · The bottleneck moves from generation to verification and review Theory

If an agent emits patches faster than people can inspect them, human review is the rate-limiter. Bhati calls this "the economics of attention" Bhati '26; Koch names the resulting "verification debt" Koch '26; Monperrus pushes it to the limit, arguing human code review should be superseded Monperrus '26; Alenezi: "the binding constraint is shifting from how fast code can be produced to how reliably it can be trusted" Alenezi '26. → Verification & gates

3 · Structure beats chat: implement from a reviewed brief, and ground intermediate artifacts Proposal

A consistent design pattern: separate exploratory conversation from execution, and validate the spec/plan artifacts against reality before generating code. Koch's "conversation-to-contract" gate — "Do not let an agent implement from a long chat. Let it implement from a reviewed brief" Koch '26; Taghavi's pre/post-phase context-grounding hooks against "context blindness" Taghavi+ '26; de Macedo's "from prompt to contract" convergence de Macedo '26; Hasanli's phase-ordered validation gates Hasanli+ '26. → Spec-first

4 · Acceptance by evidence, not plausibility — with risk-tiered gates Proposal

Output should be accepted because it satisfies evidence appropriate to its risk, not because it "looks right." Koch's evidence bundles and R0–R3 risk gates Koch '26; Hasanli's deterministic engine that gates every mutation Hasanli+ '26; Grynets' hidden-oracle security tests Grynets+ '26; Suresh's fuzzing-as-pseudo-oracle with counterexample refinement Suresh+ '26. → Verification & gates

5 · The lifecycle is being re-mapped phase-by-phase around agents Definition Proposal

Several papers re-draw the SDLC explicitly. Bhati's "Agentic SDLC (A-SDLC)" maps each classical phase to an agent equivalent with humans at intent/review/approval gates Bhati '26; Mandl's AI-assisted→integrated→driven staging around a "harness" Mandl+ '26; Li extends automation upstream from requirements into architecture Li+ '26. → The agentic SDLC

6 · Process maturity is uneven and largely unmeasured Theory

The tooling has outrun the evidence. de Macedo finds no SDD framework strong across all six process dimensions, and that "specification" is the saturated common denominator while roles/validation/portability diverge de Macedo '26; many corpus contributions are explicitly "preliminary" Hasanli+ '26 or position syntheses Koch '26 Monperrus '26, and the call for process-level benchmarks recurs. → Evidence
