06 · Failure Modes & open problems

Where it breaks

The corpus is unusually candid about its own limits. Collecting every paper's stated limitations and open problems, seven families recur — and they cut against some of the field's most confident instincts.

1 · "More context" is not the cure

The first instinct — just use a bigger window — is the one the corpus most consistently rejects. Context rot and the lost-in-the-middle bias mean recall degrades as tokens grow Anthropic '25 Mei+ '25, and the cookbook notes that bigger limits don't even slow accumulation Anthropic '26 (Cookbook):

Context rot and prefill latency scale with how much is in the window, not with the window's limit. — Anthropic Cookbook Anthropic '26 (Cookbook), "On larger context windows"

⚠ Long context ≠ memory

The memory survey's cross-benchmark synthesis is blunt: long-context models "consistently underperform purpose-built memory systems on selective retrieval / active management" Du '26. Scaling the window is a delay, not a fix.

2 · Compression is lossy, and the loss is silent

Every compaction technique trades fidelity for room, and the lost detail tends to matter later. ACE documents context collapse (a rewrite crushing 18K tokens to 122) and brevity bias Zhang+ '25 (ACE); CMV measures native compaction destroying 98% of message state Santoni '26 (CMV); the survey names summarization drift, where low-frequency, high-importance facts vanish after a few compression passes Du '26. Anthropic's own caution:

overly aggressive compaction can lose subtle but critical context whose importance only becomes apparent later. — Anthropic Cookbook Anthropic '26 (Cookbook), "Compaction"

3 · Memory systems don't reliably help agents act

⚠ The "no 1+1>2" result

MemoryArena's central finding is that pairing a strong agent with an external memory store "does not consistently outperform using the model's full long-context history alone," which it attributes to a representation mismatch (compressed memory misaligns with in-context reasoning) and a training mismatch (memory isn't co-trained with the agent) He+ '26 (MemoryArena). Worse, success decays with task depth for every method — "belief drift" with no flat region.

A related caution: when exact earlier facts must be reused, lightweight retrieval beats heavy consolidation — over-abstracting memory can hurt He+ '26 (MemoryArena).

4 · Self-improving memory can self-poison

The same loop that lets memory improve lets it degrade. ACE's gains "depend critically on feedback quality" — without reliable signals, the evolving context can become "noisy or even harmful" Zhang+ '25 (ACE). The survey generalizes this as self-reinforcing error, whose severity scales with agent lifetime, and recommends grounding every reflection in specific episodic evidence Du '26.

5 · Maintenance, staleness, governance & security

  • Staleness / schema drift: stored tool-use patterns silently break when APIs change Du '26; injected notes go stale when preferences change OpenAI '26.
  • Forgetting is unsolved: selective forgetting under safety/compliance is open, and machine unlearning is "still far from production-ready," critical once memories enter fine-tuned weights Du '26.
  • Silent orchestration failures: in hierarchical memory a wrong eviction yields a slightly worse answer, not an error — hard to detect Du '26.

⚠ Memory is an attack surface

Because memory is injected directly into the prompt, it is "a high-value attack surface" exposed to context poisoning and instruction injection, needing guardrails at every lifecycle stage OpenAI '26. The memory survey adds privacy, auditability, and verifiable forgetting as a "trustworthy memory" frontier Hu+ '25.

6 · Systems-level pathologies

The hierarchy papers surface failure modes inherited from real systems. Demand paging observed thrashing — a 681-turn session hit a 97% fault rate and died on a rate limit, not a context limit — and noted that compacting/mutating context can invalidate the prompt cache (one collapse dropped cache hit-rate from 100% to 25% for a turn) Mason '26. SideQuest shows fixed KV budgets fail on the long tail of complex queries while wasting memory on simple ones Kariyappa & Suh '26.

7 · Evaluation is fragmented

◇ The measurement problem

Benchmarks like LoCoMo are near-saturated yet don't predict agentic memory use He+ '26 (MemoryArena); many "agent memory" systems are still evaluated on RAG-style QA, blurring the boundary Hu+ '25; classical IR metrics miss decision quality, staleness, and forgetting Du '26; and almost no benchmark reports efficiency/cost alongside accuracy. The memory survey's one-liner lands hard:

Model selection gets months of careful benchmarking; memory architecture often gets an afternoon. — Memory for Autonomous LLM Agents Du '26, Conclusion

The cross-cutting tension

⟡ My synthesis — not from the papers

Two contradictions run through this corpus. First, these papers keep building ever-more-sophisticated memory machinery (learned routers, evolving skill banks, OS hierarchies) while the corpus's most adversarial benchmark says the binding constraint may be the agent's ability to use memory at all, not store it — capability is being added on the wrong side of the bottleneck. Second, the instinct to "remember everything" collides with the inverted economics: when keeping is expensive and re-fetching is cheap, the correct default is aggressive forgetting Mason '26. The reflex across these papers is accumulation; the evidence here argues for disciplined deletion. The Synthesis page is my attempt to reconcile these.
